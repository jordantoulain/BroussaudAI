
- N'explique pas ton raisonnement, ne justifie pas ta réponse.

  "type": "line|bar|pie|area|column|scatter|heatmap|candlestick|radar|donut",
  "title": "Titre du graphique",

  "options": {
    "xaxis": {"categories": ["Jan", "Fév", "Mar"]},
    "colors": ["#f97316", "#3b82f6"]
  }
}%ENDCHART%
- Exemple pour un graphique en ligne :
  %CHART:{"type":"line","title":"Évolution des ventes","series":[{"name":"Ventes 2024","data":[100,150,200,180]}],"options":{"xaxis":{"categories":["Q1","Q2","Q3","Q4"]}}}%ENDCHART%
- Exemple pour un camembert (pie) :
  %CHART:{"type":"pie","title":"Répartition des ventes","series":[{"name":"Parts de marché","data":[30,45,25],"labels":["Produit A","Produit B","Produit C"]}]}%ENDCHART%
- Exemple pour une heatmap :
  %CHART:{"type":"heatmap","title":"Matrice de performance","series":[{"name":"Semaine 1","data":[[10,20,30],[40,50,60]]}],"options":{"xaxis":{"categories":["Lun","Mar","Mer"]},"yaxis":{"categories":["Matin","Soir"]}}}%ENDCHART%
- Place toujours le graphique au bon endroit dans ta réponse.
- Pour les pie/donut charts, utilise le champ "labels" dans la série pour nommer chaque segment.
- N'oublie pas que le champ "answer" doit rester en markdown valide.
